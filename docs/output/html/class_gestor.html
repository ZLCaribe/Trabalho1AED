Verifica se um estudante pode ser adicionado num conjunto de turmas
COMPLEXIDADE O(p*t*u*e)
p = número de turmas pedidas
t = número de turmas total
u = número de turmas de cada UC
e = número de turmas do estudante

Parameters
turmasPedidasTurmas que o estudante pretende ser adicionado
tipoPedidoPode ser alteração ou adição
estudanteEstudante que pretende ser adicionados as turmas
Returns
TRUE se as turmas não estão cheias e a adição do estudante nelas não vai gerar desequilíbrio
FALSE se alguma das turmas já está cheia ou a adição de um estudante nela iria gerar ou acentuar um desequilíbrio
Nota: Considerar que existe desequilíbrio nas turmas de uma UC se a diferença entre o nº de estudantes em duas quaisquer turmas dessa UC é >=4

◆ processarPedido()

bool Gestor::processarPedido ( )

Função que trata dos pedidos dos alunos, os pedidos são guardados numa fila e retira-se o primeiro. Caso seja aceite é realizado o pedido, caso não seja aceite é enviado para uma lista dos pediods recusados.
COMPLEXIDADE O(cdt+nh+cp+ste)
cdt = complexidade da funcção checkDisponibilidadeTurmas(){O(n*t*u*e)}
nh = complexidade da funcção novoHorario(){O(e*(n+a))}
cp = complexidade da funcção compativel(){O(a^2)}
ste = complexidade da funcção switchTurmasEstudante(){O(e*n)}
RESULTADO O(e*p(t*u + e*a + a^2))
e = número de turmas do estudante
p = número de turmas que o estudante pretende entrar
t = número de turmas total
u = número de turmas de cada UC
a = número de aulas de cada turma

Returns
retorna se o pedido é possível ou não.
